# Neo4j APOC Cypher Export / Import

## Purpose

Export and import Neo4j data using Cypher scripts generated by APOC.
Useful for migrations between environments or Neo4j versions.

------------------------------------------------------------------------

## Enable APOC Export

Add to `neo4j.conf`:

    apoc.export.file.enabled=true
    apoc.import.file.enabled=true

Restart Neo4j.

------------------------------------------------------------------------

## Export Entire Database

``` cypher
CALL apoc.export.cypher.all(
  "db_export.cypher",
  {format:"cypher-shell"}
);
```

Output file is written to the Neo4j `import` directory.

------------------------------------------------------------------------

## Export Using MATCH Query

    CALL apoc.export.cypher.query(
    "MATCH (n:Person)-[r:KNOWS]->(m:Person) RETURN n,r,m",
    "persons.cypher",
    {format:"cypher-shell"}
    );

Steps performed: 1. Runs MATCH query 2. Collects nodes and relationships
3. Generates CREATE statements 4. Writes Cypher file

------------------------------------------------------------------------

## Export Subgraph

    CALL apoc.export.cypher.query(
    "
    MATCH (n)
    WHERE n.projectName='APEX'
    OPTIONAL MATCH (n)-[r]->(m)
    RETURN n,r,m
    ",
    "apex_project.cypher",
    {format:"cypher-shell"}
    );

------------------------------------------------------------------------

## Export Using apoc.export.cypher.data

    MATCH (n:APEXPage)-[r]->(m)
    WITH collect(n) AS nodes, collect(r) AS rels
    CALL apoc.export.cypher.data(nodes, rels, "apex.cypher", {})
    YIELD file
    RETURN file;

------------------------------------------------------------------------

## Import Cypher Export

    cypher-shell -u neo4j -p password -f db_export.cypher

------------------------------------------------------------------------

## Performance Optimization

    CALL apoc.export.cypher.all(
    "db_export.cypher",
    {
      format:"cypher-shell",
      useOptimizations:{type:"UNWIND_BATCH", unwindBatchSize:1000}
    });

------------------------------------------------------------------------

## Export Constraints and Indexes

    SHOW CONSTRAINTS;
    SHOW INDEXES;

Recreate them before importing data.

Example:

    CREATE CONSTRAINT FOR (n:Person) REQUIRE n.id IS UNIQUE;

------------------------------------------------------------------------

## When To Use

  Method               Purpose
  -------------------- ---------------------
  neo4j-admin dump     same-version backup
  APOC Cypher export   migrations
  CSV import           very large imports
